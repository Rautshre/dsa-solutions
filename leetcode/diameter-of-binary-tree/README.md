# Diameter of Binary Tree

    ## 💡 Intuition

The diameter of a tree is the longest path between any two nodes. This path either passes through the root of a subtree or lies entirely within one of its subtrees. The longest path through a given node is the sum of the heights of its left and right subtrees.

## 🧩 Approach

We use a recursive Depth-First Search (DFS) approach. A helper function `height` is defined, which not only calculates the height of the current subtree but also updates a global `diameter` variable. For each node, we recursively calculate the height of its left and right subtrees. The potential diameter passing through the current node is `left_height + right_height`, which we use to update our maximum `diameter` found so far. The function then returns `1 + max(left_height, right_height)` as the height of the current subtree.

## ⏱️ Complexity

- **Time:** O(N), where N is the number of nodes, because each node is visited exactly once during the DFS traversal.
- **Space:** O(H), where H is the height of the tree, due to the recursion stack. In the worst case (a skewed tree), H can be N, leading to O(N) space.

## ⚠️ Edge Cases

The solution correctly handles trees with a single node (diameter 0) and trees where the diameter does not pass through the overall root, as it considers paths through every node.
## Problem Statement

Given the root of a binary tree, return the length of the diameter of the tree.

The diameter of a binary tree is the length of the longest path between any two nodes in a tree. This path may or may not pass through the root.

The length of a path between two nodes is represented by the number of edges between them.

 
Example 1:

Input: root = [1,2,3,4,5]
Output: 3
Explanation: 3 is the length of the path [4,2,1,3] or [5,2,1,3].


Example 2:

Input: root = [1,2]
Output: 1


 
Constraints:


	The number of nodes in the tree is in the range [1, 104].
	-100 <= Node.val <= 100
