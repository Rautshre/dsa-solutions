# Valid Parentheses

    ## 💡 Intuition

The problem requires matching opening and closing brackets in a Last-In, First-Out (LIFO) manner, which is a classic use case for a stack data structure. When an opening bracket is encountered, it 'waits' for its corresponding closing bracket.

## 🧩 Approach

Iterate through the input string character by character. If an opening bracket is found, push it onto the stack. If a closing bracket is found, first check if the stack is empty; if so, it's invalid. Otherwise, pop the top element from the stack and verify if it's the correct matching opening bracket type. After processing all characters, if the stack is empty, the string is valid; otherwise, it's invalid due to unclosed opening brackets.

## ⏱️ Complexity

- **Time:** O(n), as we iterate through the string once, and stack operations (push, pop, top, empty) take O(1) time.
- **Space:** O(n), as in the worst case (e.g., '((((('), the stack could store up to n/2 characters.

## ⚠️ Edge Cases

The solution correctly handles cases like an empty stack when a closing bracket appears, or a non-empty stack at the end, indicating unclosed brackets. It also correctly identifies type mismatches (e.g., '{' closed by ']').
## Problem Statement

Given a string s containing just the characters '(', ')', '{', '}', '[' and ']', determine if the input string is valid.

An input string is valid if:


	Open brackets must be closed by the same type of brackets.
	Open brackets must be closed in the correct order.
	Every close bracket has a corresponding open bracket of the same type.


 
Example 1:


Input: s = "()"

Output: true


Example 2:


Input: s = "()[]{}"

Output: true


Example 3:


Input: s = "(]"

Output: false


Example 4:


Input: s = "([])"

Output: true


Example 5:


Input: s = "([)]"

Output: false


 
Constraints:


	1 <= s.length <= 104
	s consists of parentheses only '()[]{}'.
