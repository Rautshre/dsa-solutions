    ## 💡 Intuition

The key insight is to find the highest salary among all salaries that are strictly less than the absolute highest salary in the table.

## 🧩 Approach

First, a subquery identifies the absolute maximum salary present in the Employee table. The main query then selects the maximum salary from the Employee table, but only considering those salaries that are strictly less than the maximum salary found by the subquery. If no such salary exists (e.g., only one distinct salary), the outer MAX() function will return NULL, correctly handling the edge case.

## ⏱️ Complexity

- **Time:** O(N), because the table is scanned twice: once for the subquery to find the overall max, and once for the outer query to find the max among the remaining salaries.
- **Space:** O(1), as the query only stores a few scalar values during execution and does not require additional data structures proportional to the input size.

## ⚠️ Edge Cases

This solution correctly handles cases where there is no second highest salary (e.g., only one employee or all employees have the same salary) by returning null, as MAX() on an empty set yields null. It also implicitly handles duplicate salaries by finding the second highest *distinct* salary.
