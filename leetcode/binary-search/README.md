    ## 💡 Intuition

The core idea is to leverage the sorted nature of the array by repeatedly halving the search space. By comparing the target with the middle element, we can eliminate half of the remaining elements in each step.

## 🧩 Approach

Initialize two pointers, `low` at the start and `high` at the end of the array. While `low` is less than or equal to `high`, calculate the `mid` index. If `nums[mid]` matches the target, return `mid`. If `nums[mid]` is greater than the target, the target must be in the left half, so update `high` to `mid - 1`. Otherwise, the target must be in the right half, so update `low` to `mid + 1`. If the loop completes without finding the target, return -1.

## ⏱️ Complexity

- **Time:** O(log n), because the search space is divided by two in each iteration.
- **Space:** O(1), because only a constant number of extra variables are used.

## ⚠️ Edge Cases

The solution correctly handles cases where the target is not present, or when it's at either end of the array. It also works for arrays with a single element.
## Problem Statement

Given an array of integers nums which is sorted in ascending order, and an integer target, write a function to search target in nums. If target exists, then return its index. Otherwise, return -1.

You must write an algorithm with O(log n) runtime complexity.

 
Example 1:

Input: nums = [-1,0,3,5,9,12], target = 9
Output: 4
Explanation: 9 exists in nums and its index is 4


Example 2:

Input: nums = [-1,0,3,5,9,12], target = 2
Output: -1
Explanation: 2 does not exist in nums so return -1


 
Constraints:


	1 <= nums.length <= 104

	All the integers in nums are unique.
