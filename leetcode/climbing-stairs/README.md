    ## 💡 Intuition

This problem exhibits optimal substructure and overlapping subproblems, characteristic of Dynamic Programming. The number of ways to reach step 'n' from step 'i' depends on the number of ways to reach 'n' from 'i+1' and 'i+2'.

## 🧩 Approach

The solution uses a top-down dynamic programming approach with memoization. A recursive function `f(ind)` calculates the number of ways to reach the top (step 'n') starting from the current step 'ind'. The base cases are: if `ind == n`, return 1 (reached the top); if `ind > n`, return 0 (overshot). Otherwise, `f(ind)` is the sum of `f(ind+1)` (taking 1 step) and `f(ind+2)` (taking 2 steps). A `dp` array stores computed results to avoid redundant calculations, initialized to -1.

## ⏱️ Complexity

- **Time:** O(n), because each of the 'n' states (from 0 to n) is computed only once, and each computation takes constant time.
- **Space:** O(n), for the `dp` array to store results for 'n+1' states and for the recursion stack depth, which can go up to 'n' in the worst case.

## ⚠️ Edge Cases

The base cases `ind == n` and `ind > n` effectively handle small values of `n` (like `n=1` or `n=2`) and ensure correct termination of the recursion. No other notable tricky edge cases.
## Problem Statement

You are climbing a staircase. It takes n steps to reach the top.

Each time you can either climb 1 or 2 steps. In how many distinct ways can you climb to the top?

 
Example 1:

Input: n = 2
Output: 2
Explanation: There are two ways to climb to the top.
1. 1 step + 1 step
2. 2 steps


Example 2:

Input: n = 3
Output: 3
Explanation: There are three ways to climb to the top.
1. 1 step + 1 step + 1 step
2. 1 step + 2 steps
3. 2 steps + 1 step


 
Constraints:


	1 <= n <= 45
