## 🧩 Approach

The solution employs a top-down dynamic programming (memoization) approach. A `dp` array is initialized with -1 to mark uncomputed states. A recursive helper function `f(n, dp)` handles base cases for `n=0` and `n=1`, and if `dp[n]` is already computed, it returns the stored value. Otherwise, it recursively calculates `f(n-1, dp) + f(n-2, dp)`, stores the result in `dp[n]`, and returns it.

## ⏱️ Complexity

- **Time:** O(n), because each Fibonacci number from F(0) to F(n) is computed exactly once.
- **Space:** O(n), due to the `dp` array storing results for n+1 states and the recursion stack depth potentially going up to n.

## ⚠️ Edge Cases

The base cases F(0)=0 and F(1)=1 are explicitly handled, ensuring correct behavior for the smallest inputs. The constraints (0 <= n <= 30) prevent integer overflow or excessively deep recursion.
## Problem Statement

The Fibonacci numbers, commonly denoted F(n) form a sequence, called the Fibonacci sequence, such that each number is the sum of the two preceding ones, starting from 0 and 1. That is,

F(0) = 0, F(1) = 1
F(n) = F(n - 1) + F(n - 2), for n > 1.


Given n, calculate F(n).

 
Example 1:

Input: n = 2
Output: 1
Explanation: F(2) = F(1) + F(0) = 1 + 0 = 1.


Example 2:

Input: n = 3
Output: 2
Explanation: F(3) = F(2) + F(1) = 1 + 1 = 2.


Example 3:

Input: n = 4
Output: 3
Explanation: F(4) = F(3) + F(2) = 2 + 1 = 3.


 
Constraints:


	0 <= n <= 30
